# 01. 배열(Arrays) #

### 특징 ###  
```
- 같은 자료형의 변수로 이루어진 선형 자료구조
- 연속된 메모리 공간에 순차적으로 저장된다
- 크기는 고정(변경 불가능)
- index 부여, 0부터 시작
```  
![](https://media.geeksforgeeks.org/wp-content/uploads/array-2.png)  
출처 :   https://www.geeksforgeeks.org/array-data-structure/
  
### 사용방법 ###  
```
int[] a;                     // a : 배열의 본체를 참조하는 배열 변수
int a[];
int[] a = new int[5];        // 배열의 크기가 정해져 있을 때
int[] a = {1, 2, 3, 4, 5};   // 배열의 데이터가 정해져 있을 때
```  
  
### 시간복잡도 ###  
```
- 삽입/삭제
  * 배열 맨 앞에 삽입/삭제 : O(n)
  * 배열 맨 뒤에 삽입/삭제 : O(1)
  * 배열 중간에 삽입/삭제  : O(n)
- 탐색 : O(1)
```  
  
### 장점 ###  
```
- 인덱스 이용해 바로 접근 가능
- 연속된 메모리 공간에 존재하기 때문에 관리가 편리함
```  
  
### 단점 ###  
```
- 삽입/삭제가 어렵다 : 원소 삽입/삭제 시 해당 원소 자리 이후의 원소들을 한 칸씩 밀거나 당겨야 한다.
- 배열의 크기를 바꿀 수 없다. : 처음 생성 시 크기를 설정하기 때문.
- 연속된 메모리 때문에 공간 낭비 발생 가능
- 연속적인 메모리 할당 필요
```  
  
### 배열의 사용 ###  
```
- 데이터의 개수가 정해져 있을 때
- 데이터 삽입/삭제가 적을 때
- 검색해야 할 때
- 다차원 데이터 다룰 때
```
